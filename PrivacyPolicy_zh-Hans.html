Octavo Reader 隐私政策

最后更新: 2025年8月

隐私第一承诺: Octavo Reader 的设计将您的隐私作为重中之重。我们不收集、存储或传输任何个人数据或使用分析。

1. 我们不收集的信息

  • 个人信息: 我们不收集姓名、电子邮件地址、电话号码或任何其他个人标识符。
  • 使用分析: 我们不跟踪您如何使用本应用、您打开了哪些文档或您的阅读模式。
  • 设备信息: 我们不收集设备标识符、操作系统详情或硬件规格。
  • 位置数据: 我们不访问或收集任何位置信息。
  • 用户账户: 我们不维护用户账户或认证系统。

2. 您的文档如何被处理

本地处理

  • PDF 文档完全在您的设备上使用 Apple 的 PDFKit 框架进行处理。
  • 文本提取和 OCR (如果需要) 都在本地使用 Apple 的 Vision 框架进行。
  • 除非明确发送以进行 AI 翻译，否则文档内容绝不会离开您的设备。

AI 翻译与摘要服务

  • 当您使用 AI 翻译功能时，只有选定的文本段落会被发送到您选择的 AI 服务提供商。
  • 当您使用 AI 摘要功能时，只有提取的文本内容 (默认为10万字符，可在设置中配置最高20万) 会被发送以生成摘要。
  • 我们绝不会发送完整的 PDF 文档、图片或任何文档元数据。
  • 所有文本提取都在您的设备上本地使用 Apple 的 PDFKit 执行。
  • 您必须提供自己的 AI 服务 (OpenAI, Anthropic, Google 等) 的 API 密钥。
  • 您的 API 密钥安全地存储在您设备的钥匙串中，绝不会传输给我们。

3. 第三方 AI 服务

当您选择使用 AI 翻译或摘要功能时:

  • 选定的文本会使用您的 API 密钥直接从您的设备发送到您选择的 AI 提供商。
  • Octavo Reader 仅作为管道 - 我们不处理、存储或访问这些数据。
  • 每个 AI 提供商都有其自己的隐私政策，规定他们如何处理您发送的文本。
  • 您可以完全禁用 AI 功能，将 Octavo Reader 作为纯本地 PDF 阅读器使用。

4. 数据存储

  • 纯本地: 所有应用数据都存储在您的设备本地。
  • 无云存储: 我们不同步或备份任何数据到我们的服务器。
  • 翻译缓存: 翻译结果可能会在本地缓存以提高性能 - 这些数据绝不会离开您的设备。
  • 设置: 您的偏好和 API 密钥配置存储在您设备的安全存储中。

5. 网络使用

Octavo Reader 仅在以下情况下连接到互联网:

  • 从 arXiv 或您提供的其他 URL 下载 PDF 文档。
  • 当您明确请求翻译或摘要生成时，使用您的 API 密钥将选定的文本发送到 AI 服务。
  • 通过 App Store 检查应用更新。

6. 权限

Octavo Reader 可能会请求以下权限:

  • 文件访问: 从您的设备、iCloud 或其他文件提供商打开 PDF 文档。
  • 网络访问: 从 URL 下载 PDF 并与 AI 服务通信。
  • 不需要或请求其他任何权限

7. 儿童隐私

由于我们不收集任何个人信息，我们的应用对所有年龄段的用户都是安全的。我们不会故意收集13岁以下儿童的个人信息。

8. 数据安全

  • 所有本地数据都受到 iOS 安全功能的保护，包括设备加密。
  • API 密钥以适当的安全属性存储在 iOS 钥匙串中。
  • 没有任何数据会传输到 Octavo Reader 服务器，因为我们不运营任何服务器。

9. 您的控制权

您对自己的数据拥有完全的控制权:

  • 所有数据都存储在您的设备本地。
  • 您可以随时删除本应用以移除所有相关数据。
  • 您可以禁用 AI 功能，将 Octavo Reader 作为完全离线的 PDF 阅读器使用。
  • 您通过自己的 API 密钥配置来控制使用哪些 AI 服务。

10. 本政策的变更

如果我们对本隐私政策进行更改，我们将更新“最后更新”日期，并通过应用更新通知用户。鉴于我们不收集数据的方针，任何更改都将维持同样的隐私第一原则。

11. 联系信息

如果您对本隐私政策或我们的隐私实践有任何疑问，您可以通过以下方式联系我们:

  • App Store 评论系统
  • GitHub 仓库问题 (如果您是技术用户)
摘要: Octavo Reader 旨在成为一个完全私密的 PDF 阅读器。我们不收集任何关于您或您使用情况的数据。当您使用 AI 功能时，只有您选择的文本会使用您自己的 API 密钥发送到 AI 服务。其他一切都保留在您的设备上。
